Il problema con i prompt

"La differenza tra un prompt mediocre
e uno efficace è la struttura."

Senza struttura, l'AI riempie i vuoti da sola.
Oggi vediamo come costruirla.

01  ·  Framework base

Ogni prompt efficace è composto da quattro elementi.
Mancane uno e l'AI inventa il resto.

Card 01
Ruolo
Chi deve essere l'AI per risponderti. Più è specifico, meglio funziona.
Card 02
Contesto
La situazione, il background, i dettagli. L'AI non sa nulla di te — diglielo.
Card 03
Compito
Cosa deve fare esattamente. Non "aiutami" — di' cosa fare.
Card 04
Formato
Come vuoi la risposta: lista, paragrafo, tabella, max N righe.
Prompt strutturato

02  ·  Strutture per tool

I tre strumenti rispondono meglio a prompt
costruiti in modo leggermente diverso.

ChatGPT
Ruolo → Contesto → Task → Vincoli → Formato
System prompt vs messaggio utente ▼
Il system prompt dà le istruzioni fisse. Il messaggio utente è la richiesta puntuale.
Custom Instructions ▼
Imposta chi sei e come vuoi le risposte — attive in ogni chat senza riscriverle.
Prompt negativi ▼
Aggiungi sempre cosa NON vuoi: "niente elenchi", "nessun disclaimer", "evita termini tecnici".
Memoria conversazione ▼
ChatGPT ricorda il filo — costruisci sulla risposta invece di ricominciare da zero.
GPT specializzati ▼
Esistono GPT pre-configurati per HR, legal, marketing. Usali prima di costruire da zero.
Gemini
Contesto → Compito → Output desiderato
Integrazione Google Workspace ▼
Legge Gmail, Drive, Docs in tempo reale. Risponde sul tuo lavoro effettivo.
Gems ▼
Istruzioni persistenti — configurale una volta, funzionano in ogni conversazione.
File e multimodalità ▼
Allega immagini, PDF, Excel direttamente nel prompt. Nessun copia-incolla.
Struttura consigliata ▼
Il contesto deve venire prima del task — Gemini risponde meglio in quest'ordine.
NotebookLM ▼
Tool gemello per documenti lunghi. Carica i materiali e interrogali in linguaggio naturale.
Claude
Task → Contesto → Regole → Piano → Allineamento
Task e criteri di successo ▼
Inizia con cosa vuoi e come riconoscerai una buona risposta. Rende l'output misurabile.
Brief di successo ▼
Descrivi anche cosa NON deve essere: tono, lunghezza, cose da evitare.
File di contesto ▼
Allega documenti, brief, linee guida. Claude li usa come base per rispondere.
Regole fisse ▼
Vincoli che Claude rispetta per tutta la conversazione — definiscili all'inizio.
Allineamento pre-esecuzione ▼
Claude ti mostra il piano prima di partire. Tu approvi o correggi — nessuna sorpresa.

04  ·  Recap pratico

Le Skills

Dalla ricerca all'installazione: come aggiungere funzionalità a Claude Code senza toccare il codice.

Metodo 01 Copia e incolla in Claude Code
Step 01
Cerca
Nel browser cerca skills.sh — un catalogo di skill pronte da installare, divise per categoria e use case.
skills.sh
Step 02
Selezione
Scegli la skill che vuoi e copia il comando npx mostrato nella sua pagina di dettaglio.
npx skills add …
Step 03
Claude Code
Incolla la linea di codice nell'input di Claude Code e premi Invio. Claude installa la skill in autonomia.
incolla → invio
Step 04
Usala
Quando vuoi usarla scrivi /nome-skill e descrivi quello che vuoi creare.
/pdf → genera
Metodo 02 Metodo in Claude
Step 01
Cerca
Digita / nella chat per vedere le skills disponibili, oppure sfoglia lo store integrato. Puoi cercare per nome o per caso d'uso — es. "pdf", "commit", "review".
/ → lista skills
Step 02
Installa
Clicca su "Add skill". La skill viene salvata nel tuo profilo e diventa disponibile in tutte le conversazioni successive — nessuna configurazione extra.
~/.claude/skills/
Step 03
Usa
Richiama la skill scrivendo /nome-skill all'inizio del messaggio. Claude la riconosce, la carica e la esegue con il contesto della conversazione attiva.
/nome-skill → trigger

Il file .md

Markdown è un formato testo leggibile da umani e da macchine. È il modo più diretto per dare istruzioni persistenti a Claude.

  • Strutturare istruzioni, regole e contesto senza usare un'interfaccia grafica
  • Creare un file CLAUDE.md che Claude legge automaticamente all'avvio di ogni progetto
  • Documentare flussi di lavoro, comandi utili, vincoli e preferenze di tono
  • Rendere il contesto riutilizzabile — scrivi una volta, funziona in ogni sessione
CLAUDE.md

Il file più importante del progetto. Claude lo legge prima di ogni sessione e rispetta tutto quello che ci scrivi: tono, regole, struttura del lavoro, cosa evitare.

sintassi base
# titoli
# Titolo principale
## Sezione

# lista puntata
- Primo elemento
- Secondo elemento

# enfasi
**grassetto**    *corsivo*

# codice inline
usa il comando `/skill`
